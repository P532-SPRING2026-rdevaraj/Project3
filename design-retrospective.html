Change 1 — Multiple Diagnosis Strategies

StatusFile
NEWWeightedScoringStrategy
NEWDiagnosisStrategyConfig (strategy Map bean)
NEWStrategyType enum, EvaluationResult DTO
MODDiagnosisEngine — dispatches by rule.getStrategyType()
MODSimpleConjunctiveStrategy — filters INFERRED obs
MODAssociativeFunction — added strategy, weights, threshold

Untouched: ObservationManager, ObservationFactory, all Commands, all Decorators, all Listeners.

Unexpected touch: None. Strategy pattern absorbed the change entirely within the engine layer.

Change 2 — Decorator Processing Pipeline

StatusFile
NEWUnitValidationDecorator
NEWAnomalyFlaggingDecorator
NEWAuditStampingDecorator
NEWBaseObservationProcessor
MODObservationProcessorConfig — assembles chain
MODObservationManager — calls processor.process()
MODObservationFactory — unit validation removed

Untouched: DiagnosisEngine, all Strategies, all Commands, all Listeners.

Unexpected touch — ObservationFactory: Unit validation had to be removed from the factory and moved into UnitValidationDecorator. Redesign: The factory should only construct objects; validation should have always lived in a separate collaborator, so no factory change would be needed when decorators change.

Change 3 — Multi-User Support & Command Undo

StatusFile
NEWAuditableCommandLog — reads real user from context
NEWUndoManager — enforces same-user, single-undo
NEWObservationAwareCommand interface
MODCommandLog — captures observationId, reads user context
MODRecordObservationCommand — added undo() → REJECTED
MODRejectObservationCommand — added undo() → ACTIVE
MODCreatePatientCommand — added undo() → throws UnsupportedOp

Untouched: ObservationFactory, DiagnosisEngine, all Strategies, all Decorators, all Listeners.

Unexpected touch: None. Command interface extension (adding undo()) was confined to the command layer. AuditableCommandLog extended rather than modified the original, preserving OCP.

Change 4 — Concept Hierarchy & Propagation

StatusFile
NEWPropagationListener — infers PRESENT ancestors / ABSENT descendants
NEWObservationSource enum (MANUAL / INFERRED)
MODRuleEvaluationListener — filters MANUAL obs only
MODDiagnosisEngine — pre-filters to MANUAL before dispatch
MODObservationFactory — added createInferred() variant
MODPhenomenonTypeManager — resolves parentConcept FK

Untouched: AuditLogListener, all Commands, CommandLog, ObservationManager, all Decorators.

Unexpected touch — DiagnosisEngine: Touched to pre-filter INFERRED obs before any strategy runs. Redesign: An injectable ObservationFilter predicate passed into the engine at construction time would eliminate future engine changes when filtering rules evolve.
